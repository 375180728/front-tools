Redux源码分析

creatStore

最初会先对传入的reducer，state，enhancer进行判断，如果没有传state，默认第二个参数就是enhancer，并判断enhancer的类型，如果是函数，则返回enhancer(createStore)(reducer,state)

返回的是被加强后的createStore方法。如果enhancer和reducer不为函数，则报错

 
x
let currentReducer = reducer
let currentState = preloadedState
let currentListeners = []
let nextListeners = currentListeners
let isDispatching = false

这一步是初始化，分别保存了reducer，当前的state，以及当前订阅的函数

 
x
function ensureCanMutateNextListeners() {
    if (nextListeners === currentListeners) {
      nextListeners = currentListeners.slice();
    }
}

这个函数的作用是拷贝一个新的Listeners数组，在后面用来做其他的操作。这里用slice()方法相当于是浅拷贝，返回一个可以用来操作的新数组。而不会让nextListeners与currentListeners同时被改变

这样做的好处是可以在 dispatch的时候避免订阅和退订的操作

createStore最终主要返回了三个方法

getState

function getState() {

}

字面意思，直接返回当前的State

subscribe

 
x
function subscribe(listener) {
    if (typeof listener !== 'function') {
        throw new Error('Expected listener to be a function.');
    }
    var isSubscribed = true;
    ensureCanMutateNextListeners();
    nextListeners.push(listener);
    return function unsubscribe() {
        if (!isSubscribed) {
            return;
        }
        isSubscribed = false;
        ensureCanMutateNextListeners();
        var index = nextListeners.indexOf(listener);
        nextListeners.splice(index, 1);
    };
}

这是一个订阅函数，接受一个参数，这个参数listener即是订阅的回调函数，这个方法把listener这个回调函数加入nextListeners这个数组中。

并返回一个取消订阅unsubscribe方法，在这个函数中会先去判断isSubscribed这个变量，这个变量表示是否已经订阅。如果已经取消订阅，则返回，否则将这个变量变为false，即代表已经取消订阅。同时删除listener这个回调函数。

        throw new Error('Reducers may not dispatch actions.');
    }
​

        isDispatching = true;
        currentState = currentReducer(currentState, action);
    } finally {
        isDispatching = false;
    }

前面是一堆判断用来检测action是否有效。然后判断是否在reducer中使用dispatch。然后设置isDispatching为true，表示已经发起dispatch。然后调用reducer并且返回新的state，这个时候触发订阅的函数

然后是var listeners = currentListeners = nextListeners每一次dispatch之前都会更新存放回调函数的数组。遍历这个数组，把listener取出来并且依次调用

combineReducers

它的主要作用就是将每一个reducer合并起来。

function combineReducers(reducers) {

        }

combineReducers接受一个reducers的对象，这个对象是所有reducer的合集。首先是分别把他们的key取出。然后根据这个key，新建一个新的对象finalReducers。这个对象是经过异常处理后最终用来合并的reducers

而在这个检测的过程中会调用一个assertReducerShape，这个方法就是它会自动传入一个初始化的action，看你的reducer中有没有对他进行处理。应该返回一个state，如果没有，这个方法就会认为这是一个无效的reducer

然后就会返回一个合并的reducer名叫combination。这个combinattion最后会返回一个总的state

先从参数中取出state与action。接着同样是遍历finalReducers并且依次取出reducer并且传入对应的state与action。然后将返回的对应的新的state存在nextStateForKey中。然后依次存入nextState。即这个nextState就是最终得到的总的新的state

hasChanged这个变量是为了检测每一个循环中的nextStateForKey与previousStateForKey，即初始状态与之后的状态有没有改变。如果都没改变，hasChanged就为false，此时直接返回最初传入的state

applyMiddleware

它的返回值是一个enhancer，即为createStore的最后一个参数

在redux中，中间件的处理是action ->middleware1->middleware2... ->reducer

        middlewares[_key] = arguments[_key];
    }

            var chain = [];
​

                getState: store.getState,

                    return _dispatch(action);

            });

            });

    };

在applyMiddleware中，首先遍历把所有中间件依次取出存入middlewares对象中。然后返回一个函数，在这个函数中传入原生的createStore方法。然后继续返回一个函数，这个函数接受reducer，preloadedState，enhancer三个参数。可以看出返回的第二个函数其实就是增强后的createStore。

然后创建一个store。这个store就带有上面所说的那些方法。建立一个middlewareAPI，里面包含了store的getState和dispatch。而我们需要改造的就是dispatch这个方法。接下来遍历中间件middlewares这个数组并且依次调用middleware(middlewareAPI)，将返回值存入数组chain中。
